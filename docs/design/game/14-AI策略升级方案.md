# AI 出牌策略深度研究 — 让《天下牌》AI 更具备策略性

> 本文档分析当前 AI 实现，提出多层次改进方案，让 AI 从"最小牌优先"的新手策略升级为具备战略眼光的对手。

---

## 一、当前 AI 实现分析

文件：`src/engine/AIBrain.ts`

### 当前逻辑摘要

| 阶段 | 策略 | 评价 |
|:----:|:----:|:----:|
| **主动出牌 (play)** | 按「牌型复杂度」排序（顺子→连对→飞机→三带→对子→单张），选最小编号 | **过于简单**：只按牌型分类，不考虑手牌结构 |
| **接牌 (follow)** | 从可接牌中选「牌型最复杂、点数最小」的组合 | **可预测**：永远用最小的合法牌接 |
| **炸弹使用** | 仅当手牌 ≤ 6 或对方手牌 ≤ 4 时才用炸弹 | **阈值粗糙**：不考虑剩余血量、伤害价值等 |
| **王炸使用** | 同炸弹逻辑，没有特殊处理 | **浪费**：王炸是最高伤害，应该策略性使用 |

### 核心问题

1. **无手牌结构意识**：AI 不会评估手牌中哪些牌可以形成潜在组合，经常打散好牌（如为出一张单牌拆掉快成型的顺子）
2. **无记牌能力**：不记录已打出的牌，无法推断对手剩余手牌范围
3. **无对手建模**：不考虑对手手牌数量、剩余血量、历史出牌模式
4. **无伤害优化**：出牌时不计算伤害值，仅按牌型决定
5. **完全可预测**：每次行为都是确定性的（永远最小+最复杂），玩家可精确预测并利用

## 二、建议的分层改进方案

### 第 1 层：手牌结构分析（低实现成本，高收益）

在 AITurn 前运行手牌结构分析，评估手牌中的「组合潜能」：

```
评估维度：
- bestCombos: 当前手牌能形成的最大牌型（检测潜在于牌中的顺子/连对/飞机）
- flexibleCards: 可以被多个牌型共享的「灵活牌」（如一张 5 既可以加入 3-4-5 顺子，也可以作为单牌打出）
- criticalPairs: 构成大牌的必要配对（如两张 A 是对子的基础，若拆开打单张就失去了炸弹/三带二的潜力）
- handDensity: 手牌点数分布密度（连续点数越多，越容易形成顺子）
```

**策略影响**：
- 出牌时优先使用「灵活度最低」的牌——即只归属于一种潜在组合的牌
- 避免拆散「criticalPairs」除非没有其他选择
- 出单张时优先出「与手牌主流点数不连续」的牌，因为它们不太可能形成组合

### 第 2 层：记牌系统（中等实现成本）

维护一个 `PlayedCardTracker`：

```typescript
class CardTracker {
  playedCards: Set<Card>           // 已见到的牌
  remainingCounts: Map<Rank, number>  // 各点数剩余张数
  opponentPlayedHistory: PlayRecord[] // 对手出牌历史
  
  // 推断对手可能持有的牌型
  inferOpponentPatterns(): Pattern[]
  
  // 估算对手剩余手牌中炸弹/顺子的概率
  estimateBombProbability(): number
  
  // 估算对手的手牌「质量」
  estimateHandQuality(): number
}
```

**策略影响**：
- 如果对手已出大量 2/A/王，AI 可更放心地打大牌
- 如果某点数已有 3 张在桌上，AI 知道对手不可能有该点数的炸弹
- 对手从未出顺子 → 可能保留了顺子组合，AI 应避免出刚好可以被顺子管上的牌

### 第 3 层：策略评分引擎（中等实现成本）

核心改进：用「评分」替代「排序+取首项」的单一策略。

```typescript
interface PlayScore {
  play: HandPattern
  damage: number           // 此手造成的伤害
  cardCost: number         // 消耗的卡牌价值
  handBreakage: number     // 打散的手牌结构评分（负值）
  opponentPressure: number // 对对手施加的压力
  riskLevel: number        // 被对手接住的风险
}

function scorePlay(play, context): number {
  return (
    damage * WEIGHT_DAMAGE +
    cardCost * WEIGHT_CARD_COST +
    handBreakage * WEIGHT_HAND_BREAKAGE +
    opponentPressure * WEIGHT_PRESSURE +
    riskLevel * WEIGHT_RISK
  )
}
```

**权重设计（可调参数）**：

| 权重 | 建议初值 | 说明 |
|:----:|:--------:|:----:|
| WEIGHT_DAMAGE | 0.3 | 伤害越高越好 |
| WEIGHT_CARD_COST | -0.1 | 消耗大牌多则扣分 |
| WEIGHT_HAND_BREAKAGE | -0.2 | 打散手牌结构扣分 |
| WEIGHT_PRESSURE | 0.15 | 消耗对手手牌数加分 |
| WEIGHT_RISK | -0.25 | 被接住的风险大扣分 |

### 第 4 层：进阶策略模式（中等实现成本）

让 AI 在不同局面切换到不同策略模式：

```
策略模式：
├── 进攻模式 (offensive)
│   └─ 触发条件：对手手牌 ≤ 4 或 对手气数 < 50
│   └─ 行为：优先高伤害出牌，积极使用炸弹/王炸终结
│
├── 防守模式 (defensive) 
│   └─ 触发条件：AI 气数 < 50 或 对手手牌 ≥ AI 手牌×2
│   └─ 行为：优先保留大牌用于接牌，出小牌消耗回合
│
├── 构筑模式 (building)
│   └─ 触发条件：开局/手牌杂乱
│   └─ 行为：优先打散灵活牌和「孤牌」，保留组合潜力
│
├── 控场模式 (control)
│   └─ 触发条件：AI 持有牌权且手牌中有炸弹
│   └─ 行为：用中等牌型引出对手大牌，再用炸弹抢夺牌权
│
└── 诈唬模式 (bluff)
    └─ 触发条件：偶然触发（10% 概率）
    └─ 行为：故意出看似弱但误导性的牌（如先出单张大牌装作想快速清手牌）
```

### 第 5 层：Boss 专属策略（已有设计基础）

结合 `09-敌人系统.md` 中已经设计的 Boss 技能，为每个 Boss 定制 AI 行为：

**示例——针对玩家适配**：

| Boss | 专属 AI 行为 |
|:----:|:-----------:|
| 张角 | 优先使用黑桃牌，知道这些牌会额外结算 |
| 金兀术 | 特意出容易被玩家接住的牌，积累 x 层数后打高伤 |
| 王莽 | 优先出多张牌的牌型（利用减伤按牌数计算的机制），使用大量小牌组合 |
| 共工 | 积极追求清空手牌（触发秒杀效果）|

## 三、实现优先级建议

| 优先级 | 改进项 | 预估工时 | 效果 |
|:------:|:------|:--------:|:----:|
| **P0** | 手牌结构分析（不拆散好牌） | 2-3h | 立竿见影，消除最明显的"AI很蠢"表现 |
| **P0** | 伤害感知出牌（选伤害更高的牌） | 1-2h | AI 从被动防御变为有威胁 |
| **P1** | 策略切换模式（进攻/防守） | 3-4h | 让对局有起伏，从单调变有趣 |
| **P1** | 记牌系统 | 4-6h | 为所有高级策略提供数据基础 |
| **P2** | 评分引擎替代排序策略 | 4-5h | 统一所有策略决策 |
| **P2** | Boss 专属策略 | 每个 1-2h | 让 Boss 战真正有特色 |
| **P3** | 诈唬/控场等高级模式 | 3-4h | 让 AI 行为不可预测 |

## 四、技术实现路径

### 4.1 快速见效（P0，优先实施）

在 `AIBrain.ts` 中增加 `evaluateHandStructure()` 函数：

```typescript
function evaluateHandStructure(hand: Card[]): {
  potentialStraights: Card[][]    // 所有潜在的顺子候选
  potentialBombs: Card[][]        // 潜在的炸弹候选
  flexibleCards: Set<Card>        // 可加入多个牌型的灵活牌
  isolatedCards: Card[]           // 不属于任何组合的孤立牌
  handQuality: number             // 手牌质量评分（0-100）
}
```

在 `decidePlay()` 中，主动出牌时优先使用 `isolatedCards` 而非 `flexibleCards`。

### 4.2 伤害感知出牌（P0）

当前 AI 在主动出牌时只考虑牌型优先级。增加伤害计算：

```typescript
function calculateDamage(pattern: HandPattern): number {
  const totalPips = pattern.cards.reduce((s, c) => s + c.rank, 0)
  const coeff = getPatternCoefficient(pattern)
  return Math.floor(totalPips * coeff)
}
```

主动出牌时，在 `normalPlays` 中选择「伤害/成本比」最高的组合，而非简单的"最小编号"。

### 4.3 策略模式切换（P1）

```typescript
type StrategyMode = 'offensive' | 'defensive' | 'building' | 'control'

function determineStrategy(battleState: BattleState): StrategyMode {
  const { player, enemy, lastPlay } = battleState
  
  if (player.qi <= 50) return 'offensive'
  if (enemy.qi <= 50) return 'defensive'
  if (enemy.hand.length <= 4) return 'offensive'
  if (enemy.hand.length >= player.hand.length * 2) return 'defensive'
  
  return 'building'
}
```

## 五、示例：改进后 AI 对局场景

**改进前（当前 AI）：**
> AI 手牌：3, 4, 5, 6, 7, 8, Q, K, A, 2 → 主动出牌选「3-4-5-6-7 顺子」→ 虽然保留 8 可作为单牌，但打散了 6 张顺子的潜力

**改进后（P0+P1）：**
> AI 手牌：3, 4, 5, 6, 7, 8, Q, K, A, 2
> 手牌分析：检测到 3-4-5-6-7-8 可形成 6 张顺子（更高伤害），标记 3-8 为组合潜力卡
> 主动出牌选「Q」（孤立大牌单张）→ 保留 6 张顺子 → 若被接住则自然放弃，不伤筋动骨

**对局效果变化：**
- 旧 AI：玩家感觉"AI 不会玩，总是出小牌还打散自己的牌"
- 新 AI：玩家感觉"AI 在故意用小牌试探，会突然打出一手大顺子翻盘"

## 六、与现有系统集成建议

1. **最低侵入修改**：只在 `AIBrain.ts` 内增加新函数，不修改 `GameScene.ts` 或 `BattleTypes.ts`
2. **参数可配置**：策略权重、模式阈值、触发概率等写在 `src/config/aiConfig.ts` 中，方便数值调优
3. **扩展接口**：Boss 专属策略通过 `BossAIStrategy` 接口注入，复用通用 AI 基类
