# 游戏底层机制（事件流、触发响应机制）

---

## 1. 概述

《天下牌》采用**事件驱动架构**，各系统（对战、地图、人物、道具、经济等）通过事件总线解耦通信。所有游戏逻辑变更均以事件形式广播，各系统通过注册监听器响应感兴趣的事件。

核心设计原则：
- **单向依赖**：上层系统依赖下层，下层不感知上层
- **事件溯源**：关键状态变更有事件记录，便于调试和存档回放
- **优先级可控**：同一事件的多个响应可按优先级排序执行

---

## 2. 事件总线架构

```
┌─────────────────────────────────────────────────────────┐
│                     GameEventBus                         │
│  ┌──────────────────┐  ┌──────────────────┐              │
│  │   PriorityQueue   │  │  ListenerRegistry │              │
│  │  (HIGH/NORMAL/    │  │  {event: [fn1,   │              │
│  │   LOW 三级)       │  │   fn2, fn3...]}  │              │
│  └──────────────────┘  └──────────────────┘              │
└─────────────────────────────────────────────────────────┘
          ▲ emit(event, data)        │ on(event, fn, priority)
          │                          ▼
   ┌──────────┐              ┌──────────────┐
   │ 对战系统  │              │  人物技能系统  │
   └──────────┘              └──────────────┘
          │                          │
   ┌──────────┐              ┌──────────────┐
   │ 地图系统  │              │  道具/印记系统 │
   └──────────┘              └──────────────┘
          │                          │
   ┌──────────┐              ┌──────────────┐
   │ 经济系统  │              │  局外成长系统  │
   └──────────┘              └──────────────┘
```

- 全局唯一的 `GameEventBus` 实例挂载在 Phaser Game 对象上
- 各系统在 `create()` 阶段注册监听，`shutdown()` 阶段注销
- 事件按优先级（HIGH → NORMAL → LOW）依次分发，同优先级按注册先后

## 3. 事件定义

### 3.1 事件数据结构

```typescript
interface GameEvent<T = unknown> {
  type: GameEventType;
  data: T;
  timestamp: number;
  source: string;
  cancellable: boolean;
}
```

### 3.2 事件类型枚举

#### 一、游戏生命周期事件

| 事件类型 | 触发时机 | 携带数据 |
|---------|---------|---------|
| `GAME_INIT` | 游戏初始化完成 | `{gameConfig}` |
| `RUN_START` | 新一局开始 | `{seed, difficulty?}` |
| `RUN_END` | 一局结束 | `{result, tongbaoRemain, destinyRemain}` |
| `GAME_SHUTDOWN` | 游戏关闭 | `null` |

#### 二、地图事件

| 事件类型 | 触发时机 | 携带数据 |
|---------|---------|---------|
| `MAP_GENERATED` | 地图布局生成完毕 | `{nodes, seed}` |
| `MAP_LAYER_ENTER` | 进入新一层 | `{layer, layerType}` |
| `MAP_LAYER_COMPLETE` | 完成当前层 | `{layer, nodeResult}` |
| `NODE_ENTER` | 进入节点 | `{node}` |
| `NODE_COMPLETE` | 节点完成 | `{node, result}` |
| `PATH_CHOSEN` | 选择路径 | `{fromNode, toNode}` |

#### 三、对战事件

| 事件类型 | 触发时机 | 携带数据 |
|---------|---------|---------|
| `BATTLE_START` | 战斗开始 | `{enemy, playerQi, enemyQi}` |
| `BATTLE_END` | 战斗结束 | `{result, tongbaoReward, remainingQi}` |
| `TURN_START` | 回合开始 | `{turnNumber, activePlayer}` |
| `TURN_END` | 回合结束 | `{turnNumber}` |
| `DRAW_CARDS` | 摸牌 | `{player, cards, count}` |
| `DISCARD_CARDS` | 弃牌 | `{player, cards, reason}` |
| `PLAY_CARDS` | 出牌 | `{player, cards, pattern}` |
| `PASS_TURN` | 放弃出牌 | `{player}` |
| `HAND_CLEARED` | 手牌清空 | `{player, lastPlayedCards}` |
| `DAMAGE_PRE_CALC` | 伤害计算前（可修改） | `{source, baseDamage, multipliers}` |
| `DAMAGE_DEALT` | 伤害结算后 | `{source, target, finalDamage, cards}` |
| `QI_CHANGED` | 气数变化 | `{player, oldValue, newValue, delta, reason}` |

#### 四、经济事件

| 事件类型 | 触发时机 | 携带数据 |
|---------|---------|---------|
| `TONGBAO_EARNED` | 获得通宝 | `{amount, source}` |
| `TONGBAO_SPENT` | 消费通宝 | `{amount, purpose}` |
| `SHOP_ENTER` | 进入黄金台 | `{shopItems}` |
| `SHOP_PURCHASE` | 购买商品 | `{item, price}` |
| `CHARACTER_RECRUITED` | 招募人物 | `{character, cost}` |
| `CHARACTER_LOST` | 失去人物 | `{character, reason}` |

#### 五、道具与印记事件

| 事件类型 | 触发时机 | 携带数据 |
|---------|---------|---------|
| `DIVINATION_ACTIVATED` | 卜辞牌生效 | `{card, pattern}` |
| `BROCADE_USED` | 使用锦囊牌 | `{card, effect}` |
| `EVENT_CARD_TRIGGERED` | 事件牌触发 | `{card, result}` |
| `SEAL_APPLIED` | 施加印记 | `{card, seal}` |
| `SEAL_REMOVED` | 印记移除 | `{card, seal}` |

#### 六、人物技能触发事件（Trigger 关键词）

| 事件类型 | 触发时机 | 涉及人物示例 |
|---------|---------|------------|
| `TRIGGER_BATTLE_START` | 战斗开始时 | 姜子牙·钓璜、萧何·镇国、孔丘·万世师表 |
| `TRIGGER_TURN_START` | 每回合开始时 | 刘伯温·推背 |
| `TRIGGER_TURN_END` | 每回合结束时 | 嬴政·集权、秦桧·构陷 |
| `TRIGGER_PLAY_CARDS` | 打出牌时 | 诸葛亮·神机、关羽·武圣 |
| `TRIGGER_RECEIVE_CARDS` | 接牌时 | 荀彧·王佐 |
| `TRIGGER_ON_DAMAGE_DEALT` | 造成伤害时 | 白起·杀神 |
| `TRIGGER_ON_DAMAGE_RECEIVED` | 受到伤害时 | 李世民·天策 |
| `TRIGGER_ON_PASS` | 放弃出牌时 | 墨翟·非攻 |
| `TRIGGER_HAND_CLEARED` | 手牌清空时 | 刘邦·斩蛇 |
| `TRIGGER_QI_THRESHOLD` | 气数低于阈值时 | 岳飞·精忠、朱棣·靖难 |

## 4. 事件流详解

### 4.1 一局完整生命周期

```
GAME_INIT
  ↓
RUN_START → MAP_GENERATED
  ↓
╔══════════════════════════════════╗
║  每层循环（1-12 层）              ║
║  MAP_LAYER_ENTER                 ║
║    ├─ PATH_CHOSEN                ║
║    ├─ NODE_ENTER                  ║
║    │    ├─ [战斗] BATTLE_START    ║
║    │    │    ├─ TURN_START        ║
║    │    │    │   ├─ DRAW_CARDS    ║
║    │    │    │   ├─ PLAY_CARDS    ║
║    │    │    │   ├─ PASS_TURN     ║
║    │    │    │   └─ DAMAGE_DEALT  ║
║    │    │    └─ TURN_END          ║
║    │    ├─ BATTLE_END             ║
║    │    ├─ [黄金台] SHOP_ENTER     ║
║    │    └─ [事件] 事件牌触发       ║
║    └─ NODE_COMPLETE               ║
║  MAP_LAYER_COMPLETE               ║
╚══════════════════════════════════╝
  ↓
RUN_END（通关 或 天命耗尽）
```

### 4.2 出牌-接牌-结算流程

```
【出牌】
TURN_START
  → DRAW_CARDS（补至17张）
  → TRIGGER_TURN_START
  → 玩家选择牌型打出
  → PLAY_CARDS (player=玩家)
  → TRIGGER_PLAY_CARDS（诸葛亮神机 +2 等）
  → DAMAGE_PRE_CALC（伤害修正管线介入）

【接牌】
  → 对方响应：
     ├─ 出大牌压过 → PLAY_CARDS (player=敌方)
     │    → 牌权维持，回到【出牌】
     ├─ 出炸弹/王炸 → PLAY_CARDS (isBomb=true)
     │    → 牌权转移，回到【出牌】（攻守互换）
     └─ 放弃 → PASS_TURN
          → TRIGGER_ON_PASS（墨翟非攻等）
          → DAMAGE_PRE_CALC
          → DAMAGE_DEALT
          → TRIGGER_ON_DAMAGE_DEALT（白起+5 等）
          → TRIGGER_ON_DAMAGE_RECEIVED（李世民天策等）
          → QI_CHANGED
          → 检查气数耗尽 → BATTLE_END / 继续

【清空手牌特殊流】
PLAY_CARDS（正好出完）
  → HAND_CLEARED
  → TRIGGER_HAND_CLEARED（刘邦斩蛇 ×1.5）
  → DAMAGE_PRE_CALC（×5）
  → DAMAGE_DEALT
  → 清空方重新摸 17 张
  → 牌权转移给对方

【获得牌权补牌（安全网）】
任意一方获得牌权（新一轮出牌）时：若其手牌为空，立即摸满 17 张。
（覆盖：黄天/猛攻等弃置类技能清空手牌后，不立即重摸，待获得牌权时由本规则补满。）
```

### 4.3 伤害计算管线

```
原始伤害 = Σ(牌面点数) × 牌型系数

  ↓ Layer 1: 卜辞牌修正（局内永久，最先应用）
  │   ├─ 破军·单张 → 系数+0.5
  │   ├─ 天威·炸弹 → 伤害×1.5
  │   └─ ...
  ↓ Layer 2: 人物技能被动修正
  │   ├─ 李广·飞将 → 单张点数+3
  │   ├─ 岳飞·精忠（气数<30%）→ 伤害+30%
  │   └─ ...
  ↓ Layer 3: 印记效果修正
  │   ├─ 锋刃印 → 该牌+3
  │   └─ ...
  ↓ Layer 4: 敌人技能修正
  │   ├─ 张角 → 黑桃牌额外结算一次
  │   ├─ 王莽 → 受到伤害-牌数
  │   └─ ...
  ↓ Layer 5: 事件牌/锦囊牌效果
  │   ├─ 赤壁之战 → 伤害+50%
  │   ├─ 背水一战 → 伤害+100%
  │   └─ ...
  ↓ Layer 6: 清空手牌判定（最终 ×5）
  ↓ Layer 7: 伤害下限锁定（Min = 1）
  ↓
emit(DAMAGE_DEALT, { finalDamage })
```

**修正器优先级原则**：
- 按 Layer 顺序依次应用
- 同 Layer 内：加法修正先于乘法修正
- 多个乘法修正：累乘叠加（非累加）

## 5. 触发响应机制

### 5.1 触发器模型

所有条件性效果采用统一的 **Trigger → Condition → Response** 模型：

```
┌──────────┐     ┌──────────────┐     ┌──────────────┐
│  Trigger  │────▶│  Condition   │────▶│   Response   │
│  事件监听  │     │  条件判断     │     │  效果执行     │
└──────────┘     └──────────────┘     └──────────────┘
```

### 5.2 触发器类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **事件触发器** | 监听特定游戏事件 | `on(PLAY_CARDS, ...)` |
| **状态触发器** | 某状态值越过阈值 | `on(QI_CHANGED, qi < 30%)` |
| **回合触发器** | 固定回合数触发 | `on(TURN_START, turn === 5)` |
| **首次触发器** | 本场/本局仅一次 | `firstTime(DAMAGE_RECEIVED)` |
| **次数触发器** | 累计 N 次后触发 | `after(PASS_TURN, 2)` |

### 5.3 触发器示例

关羽·武圣——打出的单张牌，系数+0.3：
```typescript
registerTrigger({
  id: "guan_yu_skill",
  event: PLAY_CARDS,
  condition: (e) => e.player === "player" && e.pattern.type === "SINGLE",
  response: (e) => { e.pattern.multiplier += 0.3; },
  priority: Priorities.CHARACTER_SKILL,
});

```

刘邦·斩蛇——每场战斗首次清空手牌，最终伤害×1.5：
```typescript
registerTrigger({
  id: "liu_bang_skill",
  event: HAND_CLEARED,
  condition: (e) => e.player === "player",
  response: () => addExtraMultiplier("HAN_CLEAR", 1.5),
  maxTriggers: 1,
  resetOn: BATTLE_START,
});
```

## 6. 优先级体系

```
PRIORITY_IMMEDIATE = 0     // 系统级（状态锁、防死循环）
PRIORITY_CRITICAL  = 100   // 关键技能（免死判定等）
PRIORITY_HIGH      = 200   // 伤害修改器、增减伤技能
PRIORITY_NORMAL    = 300   // 普通人物技能、印记效果
PRIORITY_LOW       = 400   // UI 更新、日志记录
PRIORITY_DEFERRED  = 500   // 动画/特效触发（延迟执行）
```

**优先级应用示例**——玩家打出王炸，敌方有张角：

```
PLAY_CARDS (王炸 ♠+🃏)
  ↓ PRIORITY_CRITICAL: 牌型合法性判定
  ↓ PRIORITY_HIGH: 诸葛亮·神机 → 点数+2
  ↓ PRIORITY_HIGH: 关羽·武圣 → 单牌判定（否，跳过）
  ↓ PRIORITY_NORMAL: 张角·黑桃额外结算 → ♠ 额外计一次
  ↓ PRIORITY_NORMAL: 印记/卜辞牌效果
  ↓ PRIORITY_LOW: 日志记录
  ↓ DAMAGE_DEALT: 最终伤害
```

## 7. 响应栈

事件链式响应时使用**响应栈**管理：

```
PLAY_CARDS emit
  ├─ [栈层1] 人物A技能响应 → 修改牌型系数
  │   └─ emit DAMAGE_PRE_CALC_MODIFIED
  ├─ [栈层1] 人物B技能响应 → 触发摸牌
  │   └─ emit DRAW_CARDS
  │       ├─ [栈层2] 人物C技能响应（摸牌时触发）
  │       └─ [栈层2] 完成
  └─ [栈层1] 结算
```

- 最大深度 10 层，超出抛异常并记录，防止死循环
- 每层记录完整上下文便于调试

## 8. 系统事件交互矩阵

| | 对战系统 | 地图系统 | 人物系统 | 道具系统 | 经济系统 |
|--|:---:|:---:|:---:|:---:|:---:|
| **对战系统** 发出 | — | NODE_COMPLETE | TRIGGER_* | SEAL_* | TONGBAO_EARNED |
| **地图系统** 发出 | BATTLE_START | — | — | — | SHOP_ENTER |
| **人物系统** 监听 | TRIGGER_* | — | — | — | SHOP_PURCHASE |
| **道具系统** 监听 | DAMAGE_PRE_CALC | MAP_LAYER_ENTER | — | — | — |
| **经济系统** 监听 | BATTLE_END | NODE_COMPLETE | CHARACTER_RECRUITED | — | — |

## 9. 调试与日志

- 所有 `emit()` 在开发模式同步输出 `console.debug()`
- 格式：`[EventBus] EVENT_TYPE { key: value }`
- 缓冲区保留最近 200 条事件，供调试面板回溯
- 事件重放功能：从 `RUN_START` 起逐条重放以复现场景
