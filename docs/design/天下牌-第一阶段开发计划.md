## 三、分期任务分解（共4期）

### 第1期：核心数据模型 + 基础对战系统

**目标**：实现完整的斗地主牌型对战，能在GameScene中完成玩家 vs AI一局战斗。

#### 3.1.1 创建核心数据模型 (`src/models/`)

新建以下文件：

**`src/models/types.ts`** — 所有TypeScript类型定义：

| 类型 | 说明 |
|------|------|
| `Suit` | 花色枚举：spade、club、heart、diamond、null(王) |
| `Rank` | 点数联合类型：3-10,11-13,15,20,25,30 |
| `Card` | 单张牌接口：suit, rank, rankLabel, id |
| `CardSuit` | 牌的花色文字表示 |
| `HandPattern` | 牌型枚举：single, pair, triple, tripleOne, tripleTwo, straight, doubleStraight, airplane, bomb, jokerBomb |
| `PlayedHand` | 打出的手牌信息：pattern, cards, totalPoints, damage |
| `BattleState` | 战斗全状态：player/enemy状态、牌权、回合数、阶段 |
| `PlayerState` | 玩家状态：手牌、气数、气数上限、已招募人物列表 |
| `EnemyState` | 敌人状态：手牌、气数、名称、Boss技能(可选) |
| `PhaseType` | 阶段枚举：play(出牌)、respond(接牌)、surrender(放弃判定)、result(结算) |

**`src/models/deck.ts`** — 牌堆管理：

| 函数 | 说明 |
|------|------|
| `createDeck()` | 生成54张标准扑克牌 |
| `shuffle(cards)` | Fisher-Yates 洗牌 |
| `deal(deck, count)` | 从牌堆抽count张牌 |
| `recycleDiscard()` | 弃牌堆洗回牌堆 |

**`src/models/hand-pattern.ts`** — 牌型识别与比较（核心逻辑）：

| 函数 | 说明 |
|------|------|
| `identifyPattern(cards)` | 识别传入手牌的牌型（返回HandPattern或null） |
| `compareHands(prev, current)` | 比较两手牌，current是否能管prev |
| `getMinRank(cards)` | 获取手牌中最小点数（用于比较） |
| `findValidPlays(hand, lastPlay?)` | 从手牌中找出所有合法出牌组合 |
| `calculateDamage(hand)` | 按伤害公式计算伤害值 |

牌型识别算法优先级：

1. 先判断王炸（2张 + 小王+大王）
2. 再判断炸弹（4张同点数）
3. 再判断飞机/连对/顺子（连续点数检测）
4. 再判断三带二/三带一/三张/对子/单张

#### 3.1.2 创建 AI 对手逻辑 (`src/ai/`)

**`src/ai/basic-ai.ts`**：

| 函数 | 说明 |
|------|------|
| `choosePlay(hand)` | 持有牌权时的出牌选择（优先出最小牌型） |
| `chooseRespond(hand, lastPlay)` | 接牌选择（从小到大尝试，最小合法牌优先） |
| `shouldSurrender(hand, lastPlay)` | 判定是否放弃 |
| `getAllValidResponses(hand, lastPlay)` | 列出所有合法接牌组合 |

AI策略层次：

1. **牌权方出牌策略**：
   - 优先出单张中最小牌
   - 如有对子且单张打完后出最小对子
   - 顺子/连对在手中牌数较少时优先出
   - 炸弹/王炸作为保底手段，尽量保留到对方快赢时

2. **接牌方策略**：
   - 能用最小合法牌接就接
   - 如果接牌会导致后续无牌可打，考虑放弃
   - Bomb/JokerBomb 策略：只在对方伤害超过阈值时使用

#### 3.1.3 创建 GameScene (`src/scenes/GameScene.ts`)

战斗UI布局（1280×720）：

| 区域 | 位置 | 内容 |
|------|------|------|
| 敌方信息区 | 顶部 | 敌方名称、气数条、手牌数 |
| 出牌区域 | 中上 | 双方打出的牌展示 |
| 我方信息区 | 底部 | 我方气数条、持有通宝 |
| 手牌区 | 下方 | 我方手牌（点击选择，高亮） |
| 操作按钮 | 右下 | "出牌"、"不出/放弃"、"使用锦囊" |
| 牌型提示 | 中上 | 当前所选牌型的提示 |

GameScene核心方法：

| 方法 | 说明 |
|------|------|
| `startBattle(enemyConfig)` | 初始化战斗：洗牌、发牌、设置气数 |
| `playCards(cards)` | 玩家出牌：检测合法性、记录、切换到AI接牌 |
| `aiRespond()` | AI接牌或放弃 |
| `playerRespond(cards)` | 玩家接牌（接AI的牌） |
| `resolveDamage(hand, target)` | 结算伤害：计算、扣减、显示伤害数字 |
| `checkHandClear(hand)` | 检测清空手牌，触发×5倍率 |
| `checkBattleEnd()` | 检测气数归零，处理胜负 |
| `handleWin()` | 战斗胜利：发放通宝、更新天命 |
| `handleLose()` | 战斗失败：扣减天命、检测是否整局结束 |
| `resetForNewRound()` | 清空手牌后重摸17张 |

#### 3.1.4 创建战斗流程协调器 (`src/systems/battle-manager.ts`)

将战斗流程从UI逻辑中分离：

| 方法 | 说明 |
|------|------|
| `initBattle()` | 初始化战斗状态 |
| `playerTurn()` | 玩家出牌阶段（等待玩家操作） |
| `aiTurn()` | AI出牌阶段 |
| `resolveCards(playedBy, cards)` | 解析一手牌，触发伤害 |
| `switchTurn()` | 切换牌权 |
| `isGameOver()` | 判定战斗是否结束 |
| `endBattle(result)` | 结束战斗，返回结算数据 |

#### 3.1.5 战斗UI详细设计

**气数条**：
- 使用Phaser Graphics绘制
- 两条平行条：金色条底 + 红色/绿色填充
- 上方是敌方气数条，下方是我方气数条
- 气数变化时使用tween渐变动画

**手牌展示**：
- 每张牌为圆角矩形卡片，宽65px、高90px
- 牌面显示点数和花色符号
- 花色着色：♠♥♦♣ 对应不同颜色
- 玩家点击牌时选中/取消，选中的牌上移15px
- 多张牌可通过拖拽或点击选择
- 在右上角显示"选中牌型"提示（如"顺子 6张"）

**伤害动画**：
- 打出牌后飞向目标区域
- 伤害数字从目标位置弹出（大号红色数字，渐隐）
- 清空手牌时显示全屏闪烁效果

#### 3.2.3 创建技能系统 (`src/systems/skill-engine.ts`)

技能引擎是人物系统的核心，设计为**事件驱动**：

| 组件 | 说明 |
|------|------|
| `SkillContext` | 技能执行上下文（当前战斗状态、触发时机、目标） |
| `SkillEffect` | 技能效果类型枚举：BEFORE_DAMAGE, AFTER_DAMAGE, ON_DRAW, ON_DISCARD, ON_BATTLE_START, ON_TURN_START, ON_SURRENDER |
| `SkillRegistry` | 注册所有人物技能的映射表 |
| `applySkillEffects(context)` | 在技能触发点统一执行所有生效技能 |
| `registerFigureSkill(figureId, effect)` | 为人物注册技能 |

技能触发点（hook points）：
- `onBattleStart` — 战斗开始时（如萧何+2牌、诸葛亮选牌型）
- `onTurnStart` — 每回合开始时
- `onPlay` — 出牌时（如关羽单张+0.3系数）
- `onRespond` — 接牌时
- `onDamage` — 造成/受到伤害时
- `onDraw` — 摸牌时
- `onDiscard` — 弃牌时
- `onSurrender` — 放弃时
- `onBattleEnd` — 战斗结束时（如通宝加成）

## 七、关键技术难点

### 7.1 牌型识别算法

核心难点在于从一组手牌中正确识别斗地主牌型。算法设计：

```
输入：选择的牌列表（任意顺序）
输出：（牌型, 是否合法）

步骤：
1. 按点数分组（Map<点数, Card[]>）
2. 按组大小排序
3. 检测特殊牌型：
   a. 王炸：2张，含小王+大王
   b. 炸弹：4张同点数
   c. 飞机：≥2组连续三张，可带单张/对子
   d. 连对：≥3对连续对子
   e. 顺子：≥5张连续点数
   f. 三带二：三张+一对
   g. 三带一：三张+一单张
   h. 三张：3张同点数
   i. 对子：2张同点数
   j. 单张：1张
4. 排除非法组合：2/王不参与顺子/连对/飞机
```

### 7.2 技能引擎的事件驱动设计

人物技能需要在不耦合场景代码的情况下注入到战斗流程中。

设计方案：
```typescript
// 在战斗流程的关键点插入事件触发
class BattleManager {
  executeSkillEffects(hookPoint: SkillEffect, context: SkillContext) {
    const activeFigures = this.playerState.figures;
    for (const figure of activeFigures) {
      const effect = SkillRegistry.get(figure.id);
      if (effect && effect.hookPoint === hookPoint) {
        effect.execute(context);
      }
    }
  }
}
```

### 7.3 AI决策性能

在17张手牌中找出所有合法出牌组合可能组合数较多。性能优化：
- 预计算所有合法组合并缓存（手牌变化时重新计算）
- 按牌型排序存储，决策时按顺序遍历
- 使用位运算优化点数比较

### 7.4 Phaser 4 适配

Phaser 4 的 API 与 Phaser 3 有差异，注意：
- 使用 `this.add.*` 工厂方法创建游戏对象
- 事件系统使用 `this.events.on` / `this.scene.events.on`
- GameConfig 的类型路径：`Phaser.Types.Core.GameConfig`
- 场景生命周期：constructor → init → preload → create → update
