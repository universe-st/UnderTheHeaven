# 角色技能实现方案

## 1. 架构概览（2025-06-20 重构）

### 1.1 核心设计原则

- **技能与角色解耦**：`SkillDefinition` 不包含 `characterId`，技能代码不引用任何角色名/ID
- **敌人与玩家共用同一套技能系统**：敌人技能和玩家技能使用相同的 `SkillDefinition` 对象，通过 `SkillRegistry.registerForBattle()` 按角色数据注册
- **统一动画管线**：`SkillRunner` 自动包装技能执行的发光/头像移动/抖动动画，技能代码不再直接调用 glow

### 1.2 角色 → 技能映射

映射关系定义在 `src/models/Character.ts`：

```typescript
interface CharacterAbility {
  skillId: string;   // 指向 SkillDefinition.id
  name: string;
  description: string;
}
```

### 1.3 系统组成

| 组件 | 文件 | 职责 |
|------|------|------|
| `SkillContext` | `SkillTypes.ts` | 运行时上下文（含 `sourceCharacterId` 标记触发源） |
| `SkillDefinition` | `SkillTypes.ts` | 纯技能定义（无角色绑定） |
| `SkillRegistry` | `SkillRegistry.ts` | 按角色数据注册技能 |
| `SkillEventBus` | `SkillEventBus.ts` | 发布/订阅事件总线 |
| `SkillRunner` | `SkillRunner.ts` | 绑定技能到事件总线 + 统一动画管线包装 |
| `CharacterSlotManager` | `SkillTypes.ts` | 头像发光/移动接口（由 `GameScene` 实现） |
| `SkillVisualManager` | `SkillTypes.ts` | 精简后的视觉接口（卡片缩放、治疗显示、音效） |
| `SkillVisualManagerImpl` | `SkillVisualManagerImpl.ts` | 视觉接口实现 |

## 2. 技能动画管线（SkillAnimationPipeline）

`SkillRunner` 在每次技能触发时自动执行固定流程，技能代码无需处理：

```
async executeWithAnimation(skill, ctx):
  判断 sourceCharacterId 是否为玩家角色
  ┌─ 是玩家 ─────────────────────────────┐
  │  1. glowOn(characterId)     // 300ms │
  │  2. moveToFront(characterId)         │
  ├──────────────────────────────────────┤
  │  3. await skill.execute(ctx)   // 技能业务逻辑
  ├──────────────────────────────────────┤
  │  4. glowOff(characterId)    // 400ms │
  │  5. restoreSlot(characterId)         │
  └──────────────────────────────────────┘
  ┌─ 是敌人 ───┐
  │  直接执行    │  // 跳过动画
  └─────────────┘
```

- 异常静默捕获，不阻塞后续技能
- 同一定时点的技能**串行**执行（优先级排序后用 `for...of` + `await`）

## 3. 异步事件流架构

### 3.1 Promise 工具层 (`src/utils/AnimationUtils.ts`)

```typescript
waitForDelay(scene, ms): Promise<void>
waitForTween(scene, config): Promise<void>
waitForCounterTween(scene, config): Promise<void>
animateCardsToPositions(cards, positions, dur, scene): Promise<void>
fadeOutAndDestroy(cards, dur, scene): Promise<void>
```

### 3.2 核心事件流（async/await 驱动）

```
onPlayClick() async
  → await executePlay()
    → emit ON_PLAY (SkillRunner 自动执行动画管线)
    → await animateCardsToPositions
    → check empty hand → damage settlement or respond

executePass() async
  → await showPassAnimation
  → await playDamageSettlement

aiRespond() / aiInitiatePlay() async
  → await waitForDelay(400)
  → decidePlay → animate → await playDamageSettlement

playDamageSettlement() async
  → Stage 1: await stage1RevealCards (逐牌揭示)
  → Stage 2: await stage2ShowCoefficient (系数标签 + emit ON_COEFFICIENT_REVEALED)
  → Stage 3: await stage3ApplyDamage (最终伤害 + emit AFTER_DAMAGE)
```

## 4. 技能触发时机钩子

| 时机 | 触发点 | 来源方 ID |
|------|--------|-----------|
| `ON_PLAY` | 出牌后 | 出牌方 characterId |
| `ON_DAMAGE_CALCULATED` | Stage 2 开始 | 出牌方 characterId |
| `ON_COEFFICIENT_REVEALED` | Stage 2 完成 | 出牌方 characterId |
| `AFTER_DAMAGE` | Stage 3 完成 | 出牌方 characterId |
| `ON_TURN_START` | AI 获得牌权 | enemyCharacterId |
| `ON_GAIN_TURN` | 获得牌权时 | (待实现) |
| `PASSIVE_MODIFIER` | 持续被动 | (SkillRunner 跳过) |

## 7. 添加新技能的步骤

1. 在 `src/skills/` 下创建技能文件（如 `NewSkill.ts`），定义 `SkillDefinition`（不含 `characterId`）
2. 技能 `execute` 为 `async` 函数，使用 `visuals.animateCardScale/showHeal/playSfx`
3. 在 `src/skills/index.ts` 的 `ALL_SKILL_DEFINITIONS` 数组中注册
4. 在 `src/models/Character.ts` 对应角色的 `abilities` 中添加 `{ skillId: 'new_skill_id', name: '技能名', description: '描述' }`
5. 将资源文件放入 `public/` 并在 `LoadingScene.ts` 注册加载
