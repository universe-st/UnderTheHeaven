# 角色技能实现方案

## 1. 架构概览（2025-06-20 重构）

### 1.1 核心设计原则

- **技能与角色解耦**：`SkillDefinition` 不包含 `characterId`，技能代码不引用任何角色名/ID
- **敌人与玩家共用同一套技能系统**：敌人技能和玩家技能使用相同的 `SkillDefinition` 对象，通过 `SkillRegistry.registerForBattle()` 按角色数据注册
- **统一动画管线**：`SkillRunner` 自动包装技能执行的发光/头像移动/抖动动画，技能代码不再直接调用 glow
- **全异步事件驱动**：`SkillEventBus.emit()` 为 `async`，调用方 `await` 获取技能执行完毕信号。无技能时直接返回，有技能时串行执行

### 1.2 角色 → 技能映射

映射关系定义在 `src/models/Character.ts`：

```typescript
interface CharacterAbility {
  skillId: string;   // 指向 SkillDefinition.id
  name: string;
  description: string;
}
```

### 1.3 系统组成

| 组件 | 文件 | 职责 |
|------|------|------|
| `SkillContext` | `SkillTypes.ts` | 运行时上下文（含 `sourceCharacterId`、`coefficientLabel` 等） |
| `SkillDefinition` | `SkillTypes.ts` | 纯技能定义（无角色绑定） |
| `SkillRegistry` | `SkillRegistry.ts` | 按角色数据注册技能，维护 `skillId → characterId` 映射 |
| `SkillEventBus` | `SkillEventBus.ts` | async 事件总线（`emit` 返回 `Promise<void>`） |
| `SkillRunner` | `SkillRunner.ts` | 绑定技能到事件总线 + 统一动画管线 + 按角色栏顺序执行 |
| `AnimationUtils` | `AnimationUtils.ts` | Promise 动画工具（含 `animateCoefficientUpdate` 公共系数增长动画） |
| `CharacterSlotManager` | `SkillTypes.ts` | 头像发光/移动/顺序接口（由 `GameScene` 实现） |
| `SkillVisualManager` | `SkillTypes.ts` | 卡片缩放、治疗显示、音效接口 |
| `SkillVisualManagerImpl` | `SkillVisualManagerImpl.ts` | 视觉接口实现 |

## 2. 异步事件驱动流

### 2.1 事件发射（await 模式）

```
caller（GameScene）                          SkillEventBus
    │                                            │
    ├─ await emit(ON_COEFFICIENT_REVEALED) ──────→│
    │                                            ├─ handler1(SkillRunner) → await executeTiming()
    │                                            │   ├─ sortByCharacterOrder(skills)
    │                                            │   ├─ skill1: filter? → await executeWithAnimation()
    │                                            │   │   ├─ glowOn + moveToFront
    │                                            │   │   ├─ await skill.execute()  // 技能业务逻辑
    │                                            │   │   └─ glowOff + restoreSlot
    │                                            │   └─ skill2: ... (等等)
    │                                            └─ return
    │ ←─── 所有技能执行完毕 ───────────────────────┘
    │
    └─ 继续游戏流程
```

**关键行为：**
- `emit()` 内部 `for...of` + `await`，事件处理器串行执行
- 无处理器时直接 `return`，零开销
- 调用方统一 `await emit()`，不再手动 `waitForDelay` 猜测技能时长

### 2.2 技能执行顺序

技能按**角色栏显示顺序**执行（`playerCharacterIds` 数组顺序），敌人技能排在最后。同角色多个技能按注册顺序。

`CharacterSlotManager.getCharacterOrder(characterId)` 返回角色在角色栏中的位置索引。

## 3. 技能动画管线

`SkillRunner` 在每次技能触发时自动执行固定流程：

```
async executeWithAnimation(skill, ctx):
  ownerId = registry.getSkillOwner(skill.id)
  isPlayer = slotManager.isPlayerCharacter(ownerId)
  ┌─ 是玩家 ────────────────────────────┐
  │  1. await glowOn(ownerId)           │  // 仅光晕渐入 + 呼吸循环，无晃动
  │  2. await moveToFront(ownerId)      │  // 已在首位则跳过，不耗时
  │  3. await shakeAndPulse(ownerId)    │  // 加速左右晃动 + 同时放大，晃动后缩小
  ├─────────────────────────────────────┤
  │  4. await skill.execute(ctx)        │  // 技能业务逻辑 + 自定义动画
  ├─────────────────────────────────────┤
  │  5. await glowOff(ownerId)          │
  │  6. await restoreSlot(ownerId)      │
  └─────────────────────────────────────┘
  ┌─ 是敌人 ──┐
  │  直接执行   │  // 跳过动画
  └────────────┘
```

- `glowOn` 只负责光晕渐入与呼吸循环（无晃动/缩放），加快触发节奏
- `moveToFront` 在角色已在首位（idx≤0）时立即返回，不消耗时间
- `shakeAndPulse` 为技能触发强调动效：晃动（30ms/步，加速）与放大（→1.15）并行，晃动结束后缩小回 1.0

- 异常静默捕获（`console.warn`），不阻塞后续技能和游戏流程
- 同一定时点的技能**串行**执行

## 4. 系数增长公共动画

`animateCoefficientUpdate()` 在 `src/utils/AnimationUtils.ts` 中定义，供需修改系数的技能（如韩信点兵）调用：

```typescript
async function animateCoefficientUpdate(
  scene, labelText, typeLabel, fromCoeff, toCoeff, duration
): Promise<void>
```

- 使用 CounterTween 将系数标签数字从 `fromCoeff` 渐增到 `toCoeff`
- 技能通过 `ctx.coefficientLabel` 获取 GameScene 创建的系数标签引用
- 整数系数显示为整数，非整数保留一位小数

### 韩信「点兵」完整动画流程

```
Stage 2 显示系数标签 "✖️ 1.5（顺子）"
  → 计数器从 sumRanks 动画到 baseCoeffDamage
  → await emit(ON_COEFFICIENT_REVEALED)
    → SkillRunner 自动执行动画管线:
      → 韩信头像发光 + 移动到首位
      → 点兵技能执行:
          → 计算 newCoefficient = baseCoefficient × suitCount
          → 播放技能触发音效
          → 花色牌放大动画 (200ms)
          → await animateCoefficientUpdate: 系数标签 "1.5" → "4.5" (800ms)
          → 更新 damageInfo.coefficient / finalDamage
      → 韩信头像停止发光
  → 继续 Stage 3（系数标签渐出 → 计数增长到 finalDamage）
```

同机位后续技能（如南蛮军藤甲）会依次触发，每个都有独立的发光/移动/执行/关闭周期。

## 5. 完整游戏事件流

伤害模型：**单牌计分（sumRanks）→ 系数（coefficient）→ 相乘得 finalDamage**。
计分类技能（如丹心）在 Stage 1 修改 sumRanks；系数类技能（如点兵）在 Stage 2 修改 coefficient。二者独立，互不覆盖。

```
playDamageSettlement() async
  → Stage 1: await stage1RevealCards
      逐牌揭示：卡牌放大缩小 + 弹出 "+rank" 分数
      每张牌在分数出现后、消失前：
      → await emit(ON_SINGLE_CARD_SETTLEMENT)  // 文天祥丹心：红桃牌计分 +10，弹出分数渐增
      技能加成累加进 sumRanks；Stage 1 结束后重算 finalDamage = sumRanks × coefficient
  → Stage 2: await stage2ShowCoefficient
      计数器左移 → 系数标签渐入 → 计数器动画到 baseCoeffDamage
      → await emit(ON_COEFFICIENT_REVEALED)   // 韩信点兵/南蛮军藤甲（修改 coefficient，重算 finalDamage）
  → Stage 3: await stage3ApplyDamage
      系数标签渐出 → 计数器动画到 finalDamage → 飞向血条 → 扣血
      → await emit(AFTER_DAMAGE)             // 牛皋猛攻/强盗剪径

executePlay() async
  → await emit(ON_PLAY)                      // 李时珍本草
  → await animateCardsToPositions
  → check empty hand → damage settlement or AI respond

aiInitiatePlay() async
  → await emit(ON_TURN_START)                // 黄巾军黄天
  → await waitForDelay(400)                  // AI 思考
  → decidePlay → animate → damage settlement
```

## 8. 主动技系统

### 8.1 概述

主动技（Active Skill）是一种**不通过事件总线触发**的技能类型，而是通过 UI 按钮由玩家主动发动。其定义类型为 `ActiveSkillDefinition`。

### 8.2 ActiveSkillDefinition 接口

```typescript
interface ActiveSkillDefinition {
  id: string;
  name: string;
  description: string;
  maxUses: number;     // 最大使用次数，在获得牌权时重置
  cardFilter: (selectedCards: Card[]) => boolean;  // 选牌过滤器
  execute: (scene: Phaser.Scene, selectedCards: Card[]) => Promise<void>;  // 执行逻辑
  ownerCharacterId: string;
}
```

### 8.3 UI 交互

- 主动技按钮与「出牌」「不出」按钮并列居中排列
- 当选中牌通过某个主动技的过滤条件时，显示对应技能按钮（金色/紫色边框）
- 多个主动技通过过滤时，显示下拉菜单（倒三角触发）供选择
- 使用次数用完或选牌不通过过滤时，按钮不显示

### 8.4 刘伯温「筹策」技能

- **技能 ID**: `liubowen_chouce`
- **类型**: 主动技，限 1 次
- **效果**: 选择两张点数差 > 1 的牌（排除大王/小王/2，rank 在 [3,15]），消耗这两张牌，创造一张临时牌加入手牌
- **临时牌**：`isTemp: true`，无花色，具有裂痕蜘蛛网 + 黄色滤镜视觉效果，进入弃牌堆时直接销毁

### 8.5 添加新主动技的步骤

1. 在 `src/skills/` 下创建文件，导出 `ActiveSkillDefinition`
2. 在 `src/skills/index.ts` 中导出
3. 在 `GameScene.initActiveSkills()` 中注册
4. 在 `src/models/Character.ts` 对应角色的 `abilities` 中添加

## 9. 添加新技能（事件驱动类型）的步骤

1. 在 `src/skills/` 下创建技能文件，定义 `SkillDefinition`（不含 `characterId`）
2. 技能 `execute` 为 `async` 函数
3. 如需系数动画：通过 `ctx.coefficientLabel` 获取标签引用，调用 `animateCoefficientUpdate()`
4. 在 `src/skills/index.ts` 的 `ALL_SKILL_DEFINITIONS` 数组中注册
5. 在 `src/models/Character.ts` 对应角色的 `abilities` 中添加 `{ skillId, name, description }`
6. 将资源文件放入 `public/` 并在 `LoadingScene.ts` 注册加载

## 10. 弃置牌事件的公共机制（清空重摸）

「弃置手牌」是技能系统的公共事件，统一由 `src/utils/CardActions.ts` 的 `discardCardsFromHand()` 处理。该函数内置一条公共规则：

> **若一次弃置使该方手牌因此清空，则自动重摸至满（17 张，带飞入动画），再返回给调用方。**

- 这是弃置事件的固有行为，调用方无需自行处理清空重摸。
- 后续效果在重摸之后执行。例如黄巾军「黄天」：弃 1 张 → 若清空则先重摸满 17 → 随后「摸一张」再抽 1 → 结算后持有 18 张。
- 转移/偷取类（如强盗「剪径」把玩家牌夺给敌人）**不是**弃置：传 `{ refillOnEmpty: false }` 关闭重摸，避免「被掏空反而白得 17 张」的漏洞。
- 重摸复用 `drawCardsToHand()`：不够时自动重洗弃牌堆补入牌堆，并对新牌播放飞入动画。

当前技能中，仅黄巾军「黄天」可能触发清空重摸（敌人轮始可能仅持 1 张）；牛皋「猛攻」的过滤器要求对方手牌 ≥ 10，弃 1 后永不空。
