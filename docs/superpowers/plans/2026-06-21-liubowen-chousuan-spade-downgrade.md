# 刘伯温「筹算」黑桃牌降级补顺子 Bug 修复

## 日期

2026-06-21

## 问题

`tryFormStraightWithSpadeFill()` 函数第 46 行 `if (spadeCard.rank > fillRank) return null;` 禁止黑桃牌实际点数大于缺失点数（降级），导致 ♠️5 不能补 4 形成顺子 3-4-5-6-7。

## 技能描述

> 当你的顺子缺一张牌时，你可以将任意黑桃牌视为缺失的点数以补充（不得视为缺的最大的点数）

- 任意黑桃牌都可以视为缺失点数，包括降级（实际点数 > 缺失点数）
- 唯一限制：不能视为顺子中最大的点数

## Root Cause

第 46 行 `if (spadeCard.rank > fillRank) return null;` 无条件阻止所有降级场景，与技能描述矛盾。

## 修复

删除 `src/skills/LiuBoWenChouSuan.ts` 第 46 行 `if (spadeCard.rank > fillRank) return null;`
